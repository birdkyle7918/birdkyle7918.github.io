记笔记的历程
2021-03-13|

以前的我是个懒人，不喜欢记学习笔记，然而在我频繁想不起过去学过的东西后，我发觉，是时候改掉这个坏习惯了。以前没有自己的电脑，用本子记东西，过不了多长时间，就会找不到本子，里面记的内容也随之消失了，十分可惜。现在有电脑了，直接所有笔记都用电脑记，然后上传到云端，全部无纸化。

最开始啥也不懂，直接用记事本记纯文字，再后来发现了 Markdown，仿佛发现了新大陆一般。这玩意也太好用了，语法简单上手快，功能也多，代码行、图片、表格…… 可以直接渲染成网页，十分纯粹，看着就舒坦，于是我的笔记一律都使用 Markdown 来写了。

然后我开始寻找写 Markdown 的工具，就又发现了一个神器：Typora。常规的 Markdown 编辑器一般是分两个屏，左边是 Markdown 源码，右边是渲染后的界面，或者干脆只能写源代码，要用浏览器打开才能看到渲染后的样子。而 Typora 与众不同的地方在于，当我写下 Markdown 源码时，显示出来的直接就是渲染后的页面，所写即所见，干净利落。

为了实现多地备份，我采取了一种策略：先在 GitHub 上建立一个笔记仓库，clone 到本地，然后在本地的仓库中记笔记，再定期 push 到远端。当时的我，每次都隔一段时间手动 push，然后还手动上传到 Google Drive 和 OneDrive，连写脚本自动上传都不会（菜+懒）。这个策略持续了一段时间。

后来，我买了一台 MacBook，一次偶然的机会在少数派上看到了关于 Notion 的推荐，于是我就去尝试了这个 App，发现很有特色！贴近原生的 Markdown 语法，基于“方块”的编辑模式，还有其他功能，例如嵌入视频、ToDo 列表，Database 等，也都非常好用。最重点的是，学生凭借 edu 邮箱可以免费使用 professional 版本，真的很良心了。于是我从 Typora 编辑本地文件，改为了使用 Notion。

然而，在与 Notion 度过了一段“蜜月期”之后，我逐渐发现了 Notion 的缺点：它是完全基于 Web 的，也就是我所写的东西，全部被上传至云端，并没有本地保存一份，当我查看时，需要请求服务器获取页面。虽然这样方便了个人跨平台访问内容和多人跨平台协作，但是也造成了问题，如果某篇笔记内容偏多，打开时就会加载很长时间，这个体验确实不太舒服。后来我又了解到，Notion 开发团队目前还在成长阶段中，而且服务器也时不时发生故障，造成很多人无法打开笔记，以及丢失笔记更新等诸多问题，具体见 Notion 官方推特。我不否认 Notion 是个很棒的产品，但由于我用不上 Notion 关于多人协作的功能（也许这才是精髓），发挥不了它的全部实力，再加上上述缺点，我最终还是放弃了它。

再之后，我又尝试了几个其他的笔记 App，比如印象笔记、logseq，都没有达到我心中的预期，兜兜转转，我又回到了 Typora。由于 rclone 这个命令行工具让我很方便地将本地内容与云端同步，我制定了一个更好的策略，在本地写笔记，然后编写脚本，每隔 2 小时，就自动使用 rclone 将笔记上传至 Google Drive 和 OneDrive，当然，我的 iCloud 是自动上传的。这样，不需要我手动操作，就轻松达到了多地备份的目的。这个策略是我直到目前还在使用的，如果我之后想到或发现了更好的策略，也许我还会有所改变。如果有朋友看到了这篇文章，非常欢迎互相交流，关于如何记笔记以及保存笔记。当然，我知道我的文章很少人看…但我依然这样希望。
